# Deployment & Hosting Guide: Medical Lab Report Classifier

This document provides step-by-step instructions to deploy the Medical Lab Report Classifier system to the cloud for **FREE** using Netlify, Render, and Supabase.

---

## Architecture Overview

```
User Browser 
    │
    ├─► [HTTPS] ──► Netlify Static Frontend (HTML5 / JS)
    │
    └─► [CORS]  ──► Render FastAPI Backend (Docker Container with Tesseract)
                        │
                        ├─► Local temporary processing (stateless)
                        └─► [PostgREST] ──► Supabase PostgreSQL Database
```

---

## Step 1: Set Up Supabase PostgreSQL Database

1. Go to [Supabase](https://supabase.com) and log in or create a free account.
2. Create a new project (e.g., `medical-classifier`).
3. Once the database is provisioned, navigate to the **SQL Editor** in the left sidebar.
4. Click **New Query** and run the following SQL script to create the `reports` table:

```sql
CREATE TABLE reports (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    filename TEXT NOT NULL,
    upload_date TEXT NOT NULL,
    image_url TEXT,
    ocr_text TEXT,
    predicted_category TEXT,
    confidence NUMERIC,
    matched_keywords TEXT,
    user_confirmation TEXT,
    final_correct_label TEXT
);

-- Enable read/write access for anonymized clients
ALTER TABLE reports ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Allow anonymous read/write" ON reports FOR ALL TO anon USING (true) WITH CHECK (true);
```

5. Go to **Project Settings > API** and copy:
   - `Project URL` (Used as `SUPABASE_URL`)
   - `API Key` (the `anon / public` key, used as `SUPABASE_KEY`)

---

## Step 2: Deploy Backend to Render

1. Go to [Render](https://render.com) and log in or create a free account.
2. Click **New +** and select **Blueprint** (or select **Web Service**).
3. Connect your GitHub repository `https://github.com/vivek-798/medical_classifier.git`.
4. Render will automatically detect the `render.yaml` file in your repository.
5. Provide the following environment variables when prompted:
   - `SUPABASE_URL`: (Your Supabase URL)
   - `SUPABASE_KEY`: (Your Supabase Anon Key)
6. Render will automatically build the container using the project `Dockerfile` (installing Python, Tesseract OCR, OpenCV, and project requirements) and start the web service.
7. Copy the generated live service URL (e.g., `https://medical-classifier-backend.onrender.com`).

---

## Step 3: Deploy Frontend to Netlify

1. Go to the project directory locally and open `web/config.json`.
2. Replace the local fallback URL with your Render live backend URL:
   ```json
   {
     "API_BASE_URL": "https://medical-classifier-backend.onrender.com"
   }
   ```
3. Commit and push this change to your repository.
4. Go to [Netlify](https://www.netlify.com) and log in or sign up.
5. Click **Add new site > Import an existing project**.
6. Select **GitHub** and authorize access to `medical_classifier`.
7. Configure the build settings:
   - **Base directory**: `web`
   - **Build command**: (Leave empty, as it is a pure static site)
   - **Publish directory**: `.` (which points to the base `web` directory containing `index.html`, `app.js`, `style.css`, and `config.json`)
8. Click **Deploy Site**.
9. Netlify will deploy your static site and give you a live URL (e.g., `https://your-site-name.netlify.app`).
